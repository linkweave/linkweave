# Requirements Catalog

**Project:** Chainlink Bookmark Manager
**Source:** [vision.md](vision.md)
**Date:** 2026-04-13

---

## User Roles

| Role             | Description                                                                |
|------------------|----------------------------------------------------------------------------|
| User             | An individual who owns a collection and manages bookmarks, folders, and tags |
| Collection Owner | A User who created a collection and can share it with others               |

| C-009 | Offline Auth Trust Model | Offline access relies on a cached `UserInfoJson` in the browser's IndexedDB as proof of prior authentication. There is no password re-check, no session cookie validation, and no server contact. This is an accepted security trade-off: the cached data is the same data the user already saw on screen during their last online session. The cache is protected by same-origin policy and user-scoped keys (`{email}:*`). **Known edge case accepted:** if User A closes the browser tab without logging out, and User B opens the app on the same browser while the server is down, User B will see User A's cached bookmarks (read-only). This risk is deemed acceptable for a self-hosted bookmark manager. | Security | High     | Done   |
| C-010 | Offline Requires Prior Login | Offline mode can only work if the user has completed at least one successful online login in this browser. Users who have never logged in, or who explicitly logged out before the outage, will be redirected to the login page (which requires network). | Technical | High     | Done   |
